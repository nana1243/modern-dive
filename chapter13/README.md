### scope

#### 스코프란?
- 모든 식별자는 자신이 선언된 위치에 따라 다른 코드가 식별자 자신을 참조할 수 있는 유효범위가 달라진다.
- 이러한 유효범위를 스코프라고 한다.


* 함수 파라미터의 변수는 함수 몸체 내부에서만 유효하다.
```javascript
function add(x,y) {
    console.log(x, y);
    return x + y;
}
add(2,3)

console.log(x, y); // ReferenceError: x is not defined
```

* 함수 몸체 내부에서 선언된 변수는 함수 몸체 내부에서만 유효하다.
* 만약에 스코프라는 개념이 없다면 같은 이름을 갖는 변수는 충돌을 일으키므로 프로그램에서 변수명 하나밖에 사용하지 못한다.

```javascript
var x = 'global scope';

function foo() {
    var x = 'local';
    console.log(x); // local
}

foo();
console.log(x); // global scope

```

#### 13.2 스코프의 종류
- code는 global과 local 스코프로 구분된다.

- 전역 스코프: 코드 어디에서든 참조할 수 있는 스코프
- 지역 스코프: 특정 코드 블록에서만 참조할 수 있는 스코프.자신의 지역 스코프나 하위 스코프에서만 참조 가능하다.


#### 13.3 스코프 체인
- 함수는 전역에서 정의할 수도 있고, 함수 몸체 내부에도 정의 할 수 있다.
- 함수 몸체 내부에서 정의한 함수를 중첩 함수라고 하며, 이러한 중첩 함수는 자신이 정의된 함수를 outer function이라고 한다.
- 함수가 중첩이 되므로, 스코프 또한 중첩이 된다.(계층적 구조를 갖는다)

- 모든 스코프가 하나의 계층적 구조로 연결되며, 모든 지역 스코프의 최상위 스코프는 전역 스코프다. 이렇게 스코프가 계층적으로 연결된 것을 스코프 체인이라고 한다.
- 변수를 참조할때 자바스크립트 엔진은 스코프 체인을 통해 변수를 참조하는 코드의 스코프에서 시작하여 스코프 방향으로 이동하며 선언된 변수를 검색한다.
